In this project, I implemented a path tracing algorithm which simulates the interaction of light rays with primitive shapes (spheres and triangles), to allow for the rendering of 3D meshes with realistic lighting. The algorithm accounts for both direct lighting and indirect (reflected) lighting. To speed up the rendering process, I implemented several optimizations, including a bounding volume hierarchy (BVH) which utilizes the MÃ¶ller Trumbore algorithm for ray-box intersection, Monte Carlo integration to approximate continuous light by casting random rays, and adaptive sampling to avoid taking unnecessary samples that don't drastically change the picture.

Part 1: Ray Generation and Intersection

In this part, I set up basic ray generation and allowed for the intersection of rays with spheres and triangles, including the calculation of normals to allow for simple, fast shading.

Part 2: Bounding Volume Hierarchy

In this section I recursively generated a bounding volume hierarchy consisting of bounding boxes which contain smaller bounding boxes. This speeds up the rendering process because if a ray does not intersect the bounding box containing a primitive, it cannot possibly intersect that primitive. When a ray misses a bounding box, the algorithm no longer needs to test its intersection with any primitive within this bounding box. The BVH is structured like a tree: each box node has a left child and a right child. To create the left child and right child, I first split the bounding box along the median of its longest axis. If either child box contained no primitives, I recursively moved the split to the halfway point of the side which contained all the primitives. To detect ray intersection with a bounding box, I did something similar to the triangle test from the rasterizer project. A box can be formed from 6 intersecting planes: I checked whether the ray's intersections with those planes formed a valid intersection. In other words, I checked whether the maximum of the minima was equal to the minimum of the maxima.

Part 3: Direct Illumination

This was the most labor-intensive part of the project: implementing shading using ray tracing instead of surface normals. There are two ways to do this: uniform hemisphere sampling takes a Monte Carlo integration over a hemisphere of possible rays around a point on a surface, essentially choosing a random angle within this hemisphere such that each has the same probability, then sending a ray out from that point at the same angle to see if any object is blocking that point from the light. If not, it uses Lambert's cosine law to estimate the radiance at that point using the irradiance. The other way of doing this is importance sampling. Importance sampling is very similar to uniform sampling, but instead of sending rays out randomly back towards the light, the algorithm sends them out directly towards the light; this allows for point lights with an infinitely small radius, and leads to less grainy renders.


Part 4: Global Illumination

The key factor in global illumination is the at_least_one_bounce_radiance function, which uses a technique called Russian Roulette to allow rays to bounce around the scene and reflect off objects an infinite number of times, but makes it more and more unlikely each time that the ray will reflect instead of terminating - thus the name.

h2 align="middle">Part 5: Adaptive Sampling

This part was a fairly simple optimization to make rendering quicker. The idea behind adaptive sampling is that the algorithm uses the mean and variance of the samples at a given pixel to test how quickly the pixel's radiance is converging. Once the algorithm has determined the illuminance of the pixel within a 95% confidence interval, it ceases to take more samples from that pixel.
